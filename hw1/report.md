# 搜索算法作业说明

## 1. 作业内容

本作业使用 Python 实现以下内容：

- 图 `Graph`
- 树 `Tree`
- 栈 `Stack`
- 队列 `Queue`
- 深度优先搜索 `DFS`
- 广度优先搜索 `BFS`
- 一致代价搜索 `UCS`

代码建议放在 `src/` 目录，测试放在 `tests/` 目录。

## 2. 数据结构设计

### 2.1 图

图采用邻接表表示。每个节点保存若干个邻居以及边权，例如：

```python
{
    "A": [("B", 2), ("C", 1)]
}
```

这种表示方式适合搜索算法遍历邻居节点。

### 2.2 树

树使用根节点和子节点列表实现。也可以把树转换成图，再复用图搜索逻辑。

### 2.3 栈、队列、优先队列

- 栈：后进先出，适合 DFS
- 队列：先进先出，适合 BFS
- 优先队列：优先弹出累计代价最小的节点，适合 UCS

## 3. 搜索算法说明

### 3.1 DFS

- 思想：优先沿一条路径向深处搜索，走不通再回退。
- 使用数据结构：栈
- 完备性：在有限图且做重复状态检测时是完备的。
- 最优性：不是最优的。
- 时间复杂度：`O(V + E)`
- 空间复杂度：`O(V)`

### 3.2 BFS

- 思想：从起点开始，按层次逐层向外扩展。
- 使用数据结构：队列
- 完备性：完备。
- 最优性：当每一步代价相同，BFS 可以找到步数最少的路径。
- 时间复杂度：`O(V + E)`
- 空间复杂度：`O(V)`

### 3.3 UCS

- 思想：每次优先扩展当前路径总代价最小的节点。
- 使用数据结构：优先队列
- 完备性：当边代价为正时完备。
- 最优性：最优。
- 时间复杂度：`O((V + E) log V)`
- 空间复杂度：`O(V)`
